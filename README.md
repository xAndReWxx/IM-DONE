# PhysioAI Pro V2

**Realtime offline AI physiotherapy coach.**
Open your camera, stand in frame, and the system streams your
pose to a local AI pipeline that scores your posture, recommends
exercises, counts your reps, and coaches you in Arabic — all
without leaving your machine.

```
┌─ phone / tablet / laptop ─┐    ┌─ local backend (PC) ─────────┐
│  React SPA                 │    │  FastAPI + MediaPipe         │
│  • robotic eye landing     │ ◀▶ │  • posture analyzer          │
│  • live skeleton overlay   │ ws │  • exercise FSMs             │
│  • Arabic feedback (TTS)   │    │  • CSV-derived thresholds    │
└────────────────────────────┘    └──────────────────────────────┘
```

## Quick start (two terminals)

**Backend** (Python 3.11+):

```bash
cd backend
python -m venv .venv
source .venv/bin/activate          # Windows: .venv\Scripts\activate
pip install -r requirements.txt
uvicorn app.main:app --host 0.0.0.0 --port 8000 --reload
```

**Frontend** (Node 18+):

```bash
cd frontend
npm install
npm run dev                        # http://localhost:5173
```

Open `http://localhost:5173` on your laptop, or on a phone on
the same LAN at `http://<your-pc-ip>:5173`. The Vite dev server
proxies `/ws` and `/health` to the backend, so no CORS setup
needed in development.

### One-command launcher

```bash
./scripts/run_all.sh               # Linux / macOS
scripts\run_all.bat                # Windows
```

## What's where

| Folder       | What lives there                                                       |
|--------------|------------------------------------------------------------------------|
| `frontend/`  | React + Vite + TS SPA. Robotic eye landing, live session, Arabic TTS.  |
| `backend/`   | FastAPI server + embedded AI pipeline (MediaPipe Tasks API).           |
| `ai-engine/` | Dataset analysis reference. (The runtime pipeline now lives in `backend/`.) |
| `datasets/`  | Where to drop the training CSVs (optional, .gitignored).               |
| `scripts/`   | Run-both launchers for Linux/macOS/Windows.                            |
| `shared/`    | Placeholder for cross-language assets.                                 |

Each folder has its own `CLAUDE.md` with architecture notes,
flow diagrams, and a "where to look for X" cheatsheet.

## What's implemented

- ✅ Realtime camera streaming over WebSocket (browser → backend)
- ✅ MediaPipe pose detection (33 landmarks, Tasks API)
- ✅ Per-client EMA smoothing for jitter-free skeleton overlay
- ✅ Posture analyzer with **CSV-derived thresholds** (not guessed)
- ✅ Arabic coaching feedback (RTL + Web Speech TTS)
- ✅ Three exercise rep trackers: chin tuck, wall angel,
  thoracic extension — each with its own FSM
- ✅ Exercise recommendation cards tied to detected issues
- ✅ Posture gauge (0-100), telemetry strip, rep counter UI
- ✅ Mobile-friendly layout (LAN access tested)
- ✅ Tests on the backend (14 passing)
- ✅ Soft-fails if MediaPipe isn't installed (UI still iterates)

## What is *not* there (deliberately)

- ❌ Auth / signup / login
- ❌ Cloud or database
- ❌ Analytics / telemetry to a server
- ❌ Multi-user persistence

The MVP is **offline + single-session by design**, per the
project brief.

## Wire contract (frontend ↔ backend)

| Direction        | Type               | Effect                                |
|------------------|--------------------|---------------------------------------|
| Client → Server  | `frame`            | Run AI pipeline, reply `pose_result`  |
| Client → Server  | `select_exercise`  | Switch tracked exercise               |
| Client → Server  | `reset_reps`       | Zero current rep counter              |
| Client → Server  | `heartbeat`        | Keep-alive ping                       |
| Server → Client  | `connected`        | Handshake reply with config           |
| Server → Client  | `pose_result`      | Per-frame payload (landmarks, score, feedback, recs, rep state) |
| Server → Client  | `error`            | `{code, message, details?}`           |
| Server → Client  | `heartbeat`        | Pong                                  |

Pydantic models live in `backend/app/models/packets.py`.
TypeScript mirror lives in `frontend/src/lib/websocket-types.ts`.
**Keep them in lockstep.**

## Tuning posture sensitivity

Thresholds are loaded at server startup from:

```
backend/app/reference/good_posture_reference.json
```

This file was generated by analyzing the project's training
CSVs (`dataset_all_points.csv`, `data.csv`). The `rest` class
gave a clean baseline (~18° forward head, ~1.2° shoulder tilt,
~1.4° spine lean), and the thresholds are clinical-style
offsets above that baseline.

Edit the JSON and restart `uvicorn` — no code changes needed.

## Health check

```bash
curl http://localhost:8000/health
# → {"status":"healthy", "ai_ready":true, ...}
```

If `ai_ready` is `false`, MediaPipe couldn't initialize. The
backend will still run in degraded mode (empty `pose_result`),
which is useful when iterating on the UI alone.
